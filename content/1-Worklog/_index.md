* I completed a 12-week AWS Cloud internship and learning program through the AWS Study Group hands-on lab series. Throughout the process, I gradually explored core AWS services, from basic infrastructure services to data analytics and system administration solutions on the cloud platform.

* In the first weeks, I focused on becoming familiar with the AWS environment, compute, storage, and networking services such as Amazon EC2, Amazon S3, Amazon VPC, and resource management tools. Next, I practiced deploying databases with Amazon RDS, learned access management with AWS IAM, and applied security controls with AWS KMS, IAM Role, and AWS CloudTrail.

* In the next phase, I studied and practiced NoSQL services with Amazon DynamoDB, built automation solutions with AWS CLI, Lambda, and CloudWatch, and learned how to manage, monitor, and optimize AWS resources.

* In the final weeks, I focused on Data Analytics on AWS. I built a Data Lake architecture with Amazon S3, processed data with AWS Glue, queried data with Amazon Athena, and visualized data with Amazon QuickSight. I also practiced preparing data with AWS Glue DataBrew and building a Data Warehouse with Amazon Redshift for analytics and reporting needs.

* Through 12 weeks of practice, I gained a clearer understanding of cloud computing architecture, the process of deploying and operating systems on AWS, and improved my ability to use AWS Console, AWS CLI, and modern data services. These knowledge areas and experiences form an important foundation for future study, research, and career development in Cloud Computing and Data Engineering.
