During the internship and the process of learning about **Amazon Web Services (AWS)**, I researched and summarized several articles related to architecture design, application deployment, and practical experiences when working in a cloud computing environment. The blog posts below were built from my learning process, hands-on practice, and references from official AWS documentation to reinforce knowledge, strengthen system design thinking, and understand deployment principles on cloud platforms more clearly.

Each article focuses on a different topic, from system architecture design workflows and security principles to selecting suitable AWS services, practical experiences, and common mistakes when deploying applications. These knowledge areas help learners approach AWS in a practical way and apply them to future projects.

---

### [Blog 1 - From Requirements to Architecture: Designing a Web System on AWS](3.1-Blog1/)

This article presents the architecture design process for a web system on AWS, starting from analyzing system requirements to choosing suitable services such as **Amazon VPC, Amazon EC2, Amazon RDS, Amazon S3, Application Load Balancer**, and **AWS IAM**. It also explains important design decisions related to **Public/Private Subnet** planning, system security, scalability, high availability, and cost optimization when deploying applications on AWS.

---

### [Blog 2 - 7 Common Mistakes Beginners Make When Deploying Their First Web Application on AWS](3.2-Blog2/)

This article summarizes common mistakes that beginners often make when deploying applications on AWS, such as placing resources in the wrong VPC location, configuring **Security Group** rules incorrectly, using **Access Key** instead of **IAM Role**, storing data improperly, misunderstanding **Region**, **Availability Zone**, and **VPC**, and not monitoring the system with **Amazon CloudWatch**. It also provides recommendations and **AWS Best Practices** to help build systems that are secure, manageable, and scalable in practice.

---

### [Blog 3 - 7 Ways to Optimize Cost When Deploying Applications on AWS](3.3-Blog3/)

This article shares practical experience for optimizing cost on AWS, from choosing the right services and resource configurations, using **AWS Free Tier**, managing data lifecycle on **Amazon S3**, tracking cost with **AWS Budgets** and **AWS Cost Explorer**, to cleaning up unused resources. It also introduces the **Start Simple, Scale Later** principle and the **Cost Optimization** pillar in the **AWS Well-Architected Framework**, helping readers build systems that satisfy technical requirements while using budget efficiently.
